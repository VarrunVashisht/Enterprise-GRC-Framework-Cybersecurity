# Incident Response Plan

## Purpose
This plan defines how the organisation responds to
information security incidents.

## Incident Response Phases
1. Detection and reporting
2. Containment
3. Eradication
4. Recovery
5. Lessons learned

## Roles and Responsibilities
- Incident Manager: Overall coordination
- IT/Security Team: Technical response
- Compliance Manager: Regulatory notification
- Management: Decision making

## Reporting
All incidents must be recorded and reviewed.
