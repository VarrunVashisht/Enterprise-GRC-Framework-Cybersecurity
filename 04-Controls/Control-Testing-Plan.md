# Control Testing Plan

## 1. Purpose
This plan defines how security and compliance controls
are tested to ensure they operate effectively.

## 2. Scope
This plan applies to all key security and compliance controls.

## 3. Testing Approach
Controls are tested using the following methods:
- Evidence review
- Configuration review
- Interviews where required

## 4. Control Testing Schedule

| Control | Test Method | Frequency | Owner |
|-------|-------------|-----------|-------|
| MFA | Configuration review | Quarterly | IT Manager |
| Encryption | Evidence review | Quarterly | Security Team |
| Backups | Restore testing | Quarterly | IT Manager |
| Security Training | Attendance review | Annually | HR |
| Incident Response | Tabletop exercise | Annually | Compliance Manager |

## 5. Reporting
Test results are documented and reported to management.
Any control weaknesses are tracked until resolved.
